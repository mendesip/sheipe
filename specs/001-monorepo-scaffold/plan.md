## Summary

Initialise the Sheipe monorepo with two working apps — a Rails 8 API-only
backend and a Flutter mobile app — establishing the architectural patterns
(error handling, UUID PKs, auth-guard navigation, offline DB bootstrap,
Repository contracts) that all future features will build upon.
No business logic or data entities are introduced in this phase.

## Phase 0 — Research Findings

See [research.md](research.md) for full findings. Key decisions:

- **pgcrypto** (not uuid-ossp) for UUID PKs — Rails 8 standard
- **GoRouter `refreshListenable`** with `ChangeNotifier` bridge to `AuthCubit`
  for reactive auth guard
- **Drift lazySingleton** via get_it with `dispose` callback
- **AuthInterceptor** extends `Interceptor`; 401 → clear token + notify cubit
  (no refresh token in this phase)
- **rswag-rails + rswag-specs** gems; `rails g rswag:install` for initial setup

## Phase 1 — Design & Contracts

See [data-model.md](data-model.md) and [contracts/error-response.md](contracts/error-response.md).

- No data entities introduced
- Error response contract fully defined
- UUID via `pgcrypto` + `gen_random_uuid()` established as the single migration pattern
- Repository, LocalDataSource, RemoteDataSource abstract contracts defined
  as base classes in `lib/core/` — no concrete implementations
